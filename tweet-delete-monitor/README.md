# debot 删推监控

一个用于 X/Twitter 的 Chrome 插件。你可以在 X 页面上一键把指定推文加入删推监控，并在插件侧边栏里查看监控中和历史记录。

## 主要功能

- 在 `x.com` / `twitter.com` 的推文作者旁展示「监控删推」按钮。
- 点击插件图标后，从 Chrome 右侧边栏打开监控面板。
- 查看「监控中」和「历史」列表。
- 展示删推监控额度、最近检测时间、推文状态和到期时间。
- 支持取消监控。
- 推文删除后会在列表里标记为「已删除」。
- 检测异常会显示为「检测异常，待重试」，不会误标为已删除。

## 使用前准备

你需要先登录 debot：

<https://debot.ai/>

插件会使用浏览器里的 debot 登录态。未登录时，插件侧边栏会显示登录引导，点击「去 debot 登录」即可打开 debot 首页。

## 安装方式

当前版本需要以开发者模式加载：

1. 下载或克隆本项目。
2. 打开 Chrome：`chrome://extensions/`。
3. 开启右上角「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择目录：`chrome-extension/tweet-delete-monitor`。
6. 点击浏览器工具栏里的 debot 删推监控图标，确认右侧边栏可以打开。

## 如何添加删推监控

1. 登录 debot。
2. 打开 <https://x.com> 或 <https://twitter.com>。
3. 进入任意推文详情页，或在时间线中找到要监控的推文。
4. 点击推文作者旁的「监控删推」按钮。
5. 添加成功后，按钮会变为「已监控」或「监控中」。
6. 点击插件图标，可以在右侧边栏查看该推文。

## 右侧边栏说明

### 监控中

展示当前仍在有效期内的删推监控：

- 推文摘要
- 作者
- 推文 ID
- 当前状态
- 到期时间
- 最近检测时间
- 取消监控操作

### 历史

展示已删除、已过期、已取消或检测异常的历史记录。

## 额度和有效期

- 每个用户最多同时监控 20 条推文。
- 删推监控额度是独立额度，不占用其他社媒监控额度。
- 每条监控有效期为 5 天。
- 到期后会进入历史记录，并释放额度。
- 插件图标上的数字表示当前有效删推监控配置数。

## 状态说明

- `监控中`：推文正在被检测。
- `已监控`：这条推文已经在你的监控列表里。
- `已删除`：系统确认推文已删除。
- `已过期`：监控已超过 5 天有效期。
- `已取消`：用户主动取消监控。
- `检测异常，待重试`：本次检测无法确认结果，后续会继续重试。

## 常见问题

### 为什么显示未登录？

请先访问 <https://debot.ai/> 并完成登录。登录后回到 X 页面，刷新页面或重新打开插件侧边栏。

### 为什么添加失败？

常见原因包括：

- 未登录 debot。
- 删推监控额度已用完。
- 推文链接或 tweet ID 无法解析。
- 推文不存在或无法获取。
- 网络或服务临时异常。

### 为什么检测异常不是已删除？

检测异常可能来自网络错误、接口限流、权限问题或临时不可用。只有系统明确确认推文删除时，才会标记为「已删除」。

### 刷新 X 页面后，已监控状态还会保留吗？

会。插件会读取你的有效监控列表，如果当前推文已经被监控，按钮会显示「已监控」。
